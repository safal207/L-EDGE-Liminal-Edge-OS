# Meta-Orchestrator

Meta-Orchestrator – наблюдающий слой, собирающий мета-снимок всей системы: организм, поле, намерения и коллективный резонанс. Он не изменяет поведение органов, а агрегирует существующие метрики в связное состояние, чтобы внешние наблюдатели видели согласованность и напряжённость L-EDGE.

## Основные обязанности
- Читает последние состояния heartbeat, Field/Noosphere, Intent и Collective Resonance.
- Рассчитывает простые мета-показатели: уровень стресса, стабильность, согласованность и напряжённость.
- Формирует текстовый комментарий, пригодный для health/report эндпоинтов.

## API модуля
- `update(context?)` — пересчитывает снимок, используя переданные данные или провайдеры из конструктора.
- `getLastSnapshot()` — возвращает последний рассчитанный снимок или `null`, если расчёта не было.

## Интеграция
- Создаётся в `core/systemContext.ts` и обновляется на каждом heartbeat после расчёта поля, намерений и резонанса.
- Снимок публикуется через новый REST-эндпоинт `/api/system/meta/state` и может быть вложен в heartbeat.

## Ограничения
- Метрика опирается на существующие поля; если данных недостаточно, возвращается "unknown"-снимок.
- Эвристики intentionally простые и легко настраиваются.
