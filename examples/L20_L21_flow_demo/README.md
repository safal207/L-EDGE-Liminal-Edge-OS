# L20 + L21 Flow Demo

Этот пример показывает связку между сенсором потока (L21) и оркестратором решений (L20).

## Что происходит
1. **L21_FlowAlignment** получает внутреннее/контекстное/коллективное состояние и выдаёт:
   - Flow Alignment Index (FAI) и разложение по осям + качество (low/medium/high).
   - Luck Window (открытость, сила, тайминг now/soon/later).
   - Cooperative Field (score + режим solo/team/clustered).
   - Flow Suggestion (мягкий режим: stabilize/explore/push/pivot/slow_down) и объяснения.
2. **L20** принимает этот flow-state, подстраивает веса метрик RSTE под режим и выбирает решение,
   которое резонансно **и** в такт потоку.
3. Демо считает два прогона: **before flow** (базовые веса без оси удачи) и **after flow** (с учётом режима потока),
   чтобы увидеть дельту в выбранном решении и скоринге.

## Файлы
- `inputs_example.json` — пример входных данных для L21 и кандидатов для L20.
- `run_demo.ts` — запускает L21, считает решения до/после учёта потока, выводит объяснения и дельту скоринга.

## Запуск
Из корня репозитория:

```bash
npx ts-node examples/L20_L21_flow_demo/run_demo.ts
```

В выводе видно:
- активный режим потока и объяснения `flow_explain`,
- как меняется decision envelope при включении оси удачи,
- список кандидатов с резонансными скорингами до/после.
