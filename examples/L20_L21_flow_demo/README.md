# L20 + L21 Flow Demo

Этот пример показывает связку между сенсором потока (L21) и оркестратором решений (L20).

## Что происходит
1. **L21_FlowAlignment** получает внутреннее/контекстное/коллективное состояние и выдаёт:
   - Flow Alignment Index (FAI) и разложение по осям.
   - Luck Window (есть ли окно удачи и его сила).
   - Flow Suggestion (мягкий режим: stabilize/explore/push/pivot/slow_down).
2. **L20_ResonantDecisionOrchestrator** принимает этот flow-state, подстраивает веса метрик RSTE под режим, и выбирает решение, которое резонансно **и** в такт потоку.

## Файлы
- `inputs_example.json` — пример входных данных для L21 и кандидатов для L20.
- `run_demo.ts` — запускает L21, передаёт его выход в L20, выводит решение и скоринг.

## Запуск
Из корня репозитория:

```bash
npx ts-node examples/L20_L21_flow_demo/run_demo.ts
```

В выводе можно увидеть выбранный режим потока, итоговое решение и как изменились веса резонанса под этот режим.
