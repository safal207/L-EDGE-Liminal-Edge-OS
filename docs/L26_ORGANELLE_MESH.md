# L26 — Organelle Mesh & Mineral Matrix Layer

L26 моделирует органеллы клетки, цитоскелет и минеральную матрицу LIMINAL-организма.
Если L25 (CellKernel) решает, **что** делать и сколько энергии выделить, то L26 отвечает за то,
**какие органеллы возьмут работу**, как распределить нагрузку, сохранить форму организма и
поддерживать "минеральную среду" для устойчивости и регенерации.

## Зачем нужен слой L26
- держать реестр органелл и их профилей (ёмкость, энергия, здоровье);
- принимать задачи от L25/DecisionBus и сопоставлять их с подходящими органеллами;
- защищать организм от перегрузок, разносить нагрузку по профилям;
- хранить «форму» организма через простой граф связей (цитоскелет);
- поддерживать минеральную матрицу: структурную прочность, чистоту каналов, скорость микроремонта;
- строить метрики здоровья и баланса органелл, а также состояние минеральной среды.

## Био-аналогия → инженерные роли

| Органелла (био)    | Роль в L-EDGE                                                     |
|--------------------|------------------------------------------------------------------|
| Цитоскелет         | Граф связей между модулями, форма организма                      |
| Митохондрии        | Энергетические модули (CPU/GPU пулы, воркеры, кэши)              |
| Рибосомы           | Исполнение задач (job-queues, pipeline-стадии)                   |
| Аппарат Гольджи    | Маршрутизация/упаковка результатов, события, webhooks            |
| ЭПР                | Транспорт данных (шины, event-bus, data pipelines)               |
| Лизосомы           | Очистка, отмена, retries, dead-letter                            |
| Вакуоли/резервуары | Буферы ожидания, rate-limit, backpressure                        |
| Минеральная матрица| Минеральный «каркас», ионная среда, микроремонт, адаптация       |

## Что добавляет Mineral Matrix Layer
L26 усиливает организм за счёт «минеральной среды», которая поддерживает устойчивость,
проводимость сигналов и восстановление:

- **структурная прочность** — как кальций в костях или кремний в микросхемах;
- **ионная проводимость** — чистый канал синхронизации, снижает шумы для L21/L22;
- **ускоренный ремонт** — выделяет ресурс на восстановление перегруженных зон;
- **адаптация под нагрузкой** — укрепляет слабые места после стресса;
- **однородная среда** — предотвращает рассыпание высших слоёв под пиковыми нагрузками.

### Параметры минеральной матрицы
```ts
export interface MineralMatrix {
  structuralIntegrity: number;   // прочность всей архитектуры
  ionFlowStability: number;      // чистота ионных/сигнальных каналов
  microRepairRate: number;       // скорость саморемонта
  adaptationSpeed: number;       // скорость адаптации под новые нагрузки
  mineralBalance: Record<string, number>;  // карта «минералов»: Mg, Ca, Fe, Si...
}
```

## Как L26 работает вместе с L21–L27
- **L24 → L25 → L26**: L24 выбирает сценарий, L25 оценивает энергетику и собирает `CellActionPlan`,
  L26 превращает его в `OrganelleTask[]` и раздаёт органеллам, учитывая минеральную среду.
- **L27**: обновляет лимиты энергии/здоровья органелл и минеральный баланс (витамины/минералы),
  может временно бустить нужные органеллы.
- **L21–L23**: используют метрики L26 (баланс, перегрузка, структурная прочность, чистота каналов)
  для корректировок потока, вязкости информации и внутреннего нарратива.

## Метрики L26
- **avgHealth** — среднее здоровье органелл (0..1).
- **loadBalanceIndex** — равномерность нагрузки (1 — идеально ровно).
- **congestionScore** — насколько сетка забита (0 — свободно, 1 — перегруз).
- **structuralIntegrity** — прочность «каркаса» (микс health органелл и минеральной матрицы).
- **ionFlowStability** — чистота сигнальных каналов (баланс нагрузки и стабильность матрицы).
- **microRepairPotential** — способность к саморемонту (минеральный запас + текущие сбои).
