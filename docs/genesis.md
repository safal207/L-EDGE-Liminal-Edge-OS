# Genesis Seeds / SOMA Seeds

Genesis — 14-я точка сборки, в которой система начинает формировать "деток": семена будущих модулей. Это не автогенерация кода,
а карта готовности: какие живые элементы назрели из текущих состояний **Origin**, **Pathway**, **FuzzyEvolution** и подсказок **
ResonanceTuner**.

## Что возвращает план

`GenesisPlan` содержит:

- `ready` — семена, которые уже созрели и могут быть развёрнуты;
- `deferred` — семена, которые ощущаются, но предлагаются к отложенному запуску;
- `summary` — короткое резюме режима (интеграция, рост, ускорение) и количества семян.

Типы семян:

- `soma_cell` — новая клетка состояния/устойчивости;
- `metric_probe` — зонд/сенсор для тонкого измерения нагрузки;
- `resonance_agent` — агент, работающий с внешним полем;
- `memory_trace` — углубление памяти и логов;
- `integration_task` — задачка на выравнивание и интеграцию изменений.

## Как читать ready vs deferred

- **Ready**: можно брать и воплощать сразу (новая SOMA-клетка, сенсор, агент).
- **Deferred**: система чувствует их потребность, но советует дождаться более устойчивого режима.

## Мини-пример: вход → план семян

**Входные сигналы**

```json
{
  "tuning": { "mode": "steady_growth", "comment": "Поле устойчивое, можно мягко расти" },
  "fuzzy": {
    "tension": { "low": 0.15, "medium": 0.35, "high": 0.5 },
    "coherence": { "low": 0.2, "medium": 0.5, "high": 0.3 },
    "alignment": { "low": 0.1, "medium": 0.4, "high": 0.5 },
    "stringMode": "resonant"
  },
  "origin": { "mission": "Поддерживать лиминальные пространства", "stressSignals": [] },
  "pathway": { "branches": ["soma", "sensors"], "trend": "balanced" }
}
```

**Выходной `GenesisPlan`**

```json
{
  "timestamp": 1732303333000,
  "ready": [
    {
      "id": "seed_soma_stability_1",
      "kind": "soma_cell",
      "priority": "high",
      "reason": "Нужна дополнительная клетка устойчивости под умеренной нагрузкой.",
      "suggestedShape": "Локальный модуль стабилизации нагрузки и мониторинга core-показателей."
    },
    {
      "id": "seed_probe_load_1",
      "kind": "metric_probe",
      "priority": "medium",
      "reason": "Хотим точнее мерить напряжение при росте резонанса.",
      "suggestedShape": "Лёгкий сенсор нагрузки для когнитивных и IO-каналов."
    }
  ],
  "deferred": [
    {
      "id": "seed_resonance_agent_1",
      "kind": "resonance_agent",
      "priority": "medium",
      "reason": "Связи снаружи пока слабые — лучше подождать устойчивого резонанса.",
      "suggestedShape": "Агент для мягкого считывания внешнего поля (юзеры/сервисы)."
    }
  ],
  "summary": "Готовы 2 семени (стабилизация + сенсор), наружный резонанс — в отложку."
}
```

## Как использовать оператору/архитектору

1. Запросить план: `GET /api/system/genesis/plan`.
2. Выбрать семена из `ready`, материализовать их в реальные SOMA-клетки, сенсоры или агенты.
3. При необходимости зафиксировать `deferred` в бэклог — вернуться к ним, когда ResonanceTuner покажет устойчивый рост.
4. После внедрения смотреть, как это повлияло на FuzzyEvolution и последующие планы Genesis.
